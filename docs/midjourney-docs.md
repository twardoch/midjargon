# Midjourney Docs

## Chapter 1: Web Quick Start

Check out this video for a quick introduction to midjourney.com, then follow the guide below to create your first images.

### Making Images on midjourney.com

1\. **Make Images from the Imagine Bar**

At the top of most pages, you'll see the imagine bar. Type a prompt and hit enter.

Your images appear on the Create tab. Click over to see results.

2\. **Make Variations**

From the Create tab, click on any image to open the lightbox. From there, you can make new versions with variations.

The lightbox also shows additional tools: Upscale, Reframe, Repaint.

3\. **Change Your Settings**

The settings button in the imagine bar lets you set default parameters for all prompts. Adjust aspect ratio, stylization, variety, generation speed, and stealth mode.

4\. **Use Images in Your Prompts**

Midjourney can use existing images as inspiration. Drag and drop any image from the Explore page or your archive to the imagine bar.

There are three different ways to use images:

For details on reference images, see our Working with Images on Web page.

5\. **Save and Manage Images**

Visit the Archive tab to download, filter, or organize images.

Click the save button wherever you see it to save individual images. Download in bulk by clicking outside the grid and dragging to select multiple images.

See the Archive page for more information on filters and options.

6\. **Create Collaboratively with Chat**

Work together with others on the Chat tab. Find or create shared spaces to collaborate.

Check out the Daily Theme chat for daily prompt challenges. See how others incorporate the theme or create alongside them.

### Need Help?

Learn about Midjourney's features in our documentation. Some popular topics:

- Setting non-square aspect ratios
- Using images as references
- The describe feature suggests prompt words for similar images

Ask questions in the Help Room anytime.

For account issues, visit help.midjourney.com for billing FAQs or support tickets.

## Chapter 4: Prompts

A prompt is a short text phrase that the Midjourney Bot interprets to produce an image. The Bot breaks prompts into tokens compared to training data to generate results. Well-crafted prompts create unique images.

### Basic Prompts

Can be as simple as one word, phrase, or emoji.

**Prompting Tip:** 
The Bot works best with simple, short phrases. Avoid long lists of requests.

Instead of: "Show me a picture of lots of blooming California poppies, make them bright, vibrant orange, and draw them in an illustrated style with colored pencils"

Try: "Bright orange California poppies drawn with colored pencils"

### Advanced Prompts

Include image URLs, text phrases, and parameters.

**Image Prompts**
Add image URLs to influence style and content. URLs go at the prompt's beginning.

Read more about Image Prompts.

**Text Prompt**
Describe what you want to generate.

**Parameters**
Change generation behavior. Include aspect ratios, models, upscalers, etc. Go at the prompt's end.

Read more about Parameters.

### Prompting Notes

**Word Choice**
More specific synonyms work better. Instead of "big," try "huge," "gigantic," "enormous," or "immense."

**Plural Words and Collective Nouns**
Plurals leave details to chance. Try specific numbers: "Three cats" instead of "cats." Collective nouns work too: "flock of birds."

**Focus on What You Want**
Describe what you want, not what you don't want. Asking for a party "without cake" might still generate cake. Use the `--no` parameter to exclude elements.

**Prompt Length and Details**
Simple prompts work but rely on Midjourney's default style. Include important elements for more control.

Be clear about crucial context:
- Subject: person, animal, character, location, object
- Medium: photo, painting, illustration, sculpture
- Environment: indoors, outdoors, moon, underwater, city
- Lighting: soft, ambient, overcast, neon, studio
- Color: vibrant, muted, bright, monochromatic
- Mood: sedate, calm, raucous, energetic
- Composition: portrait, headshot, closeup, birds-eye

## Chapter 6: Variations

Use `V1` `V2` `V3` `V4` buttons under grids, or `Vary (Strong)` and `Vary (Subtle)` under upscaled images.

The `V` buttons correspond to grid positions like reading order.

Control default variation with `🎨 High Variation Mode` and `🎨 Low Variation Mode` in settings. High is default.

Use `/prefer variability` to toggle between modes.

Under upscaled images, `🪄 Vary (Strong)` equals High Mode, `🪄 Vary (Subtle)` equals Low Mode. These ignore personal settings.

Want to change prompts during variations? Enable `Remix Mode` in settings.

### Variation Examples

Versions 5/6 and Niji 5/6 support High/Low modes.

**High Variation Mode** (default): Varies composition, elements, colors, details. Use `V` buttons or `🪄 Vary (Strong)`.

**Low Variation Mode**: Retains composition and colors, subtle detail changes. Select in `/settings` then use `V` buttons, or use `🪄 Vary (Subtle)`.

Example: `/imagine prompt` `a cute robot holding flowers`

- Original Upscaled Image
- Image grid using `🪄 Vary (Subtle)`
- Image grid using `🪄 Vary (Strong)`

Example: `/imagine prompt` `a bird watching a butterfly`

- Original Upscaled Image
- Image grid using `🪄 Vary (Subtle)`
- Image grid using `🪄 Vary (Strong)`

## Chapter 7: Upscalers

Current models (V5, V6, Niji 5, 6) produce 1024×1024 grids. Use `U1` `U2` `U3` `U4` to separate images.

Then use `Upscale (Creative)` or `Upscale (Subtle)` to increase size.

**Upscale (Subtle)**: Doubles size, keeps details similar.
**Upscale (Creative)**: Doubles size, adds new details.

Experiment to find what suits your style.

Upscaling uses GPU minutes. Can take up to twice as long as initial generation.

Not compatible with --tile parameter in V5.x/6.0.

### How to Use Upscale Tools

1. Generate an image with `/imagine`
2. Select with `U` buttons
3. Choose `Upscale (Subtle)` or `Upscale (Creative)`

Upscale jobs generated before new upscalers by refreshing with `/show` command. Compatible upscalers appear under refreshed jobs.

### Legacy Upscalers

Earlier models generated lower-resolution grids. Legacy upscalers increase size and add details.

Multiple legacy models available for older images. Uses GPU minutes.

The Bot processes jobs on high-end GPUs. Each processing minute counts as one GPU minute. Fast Mode has limited GPU minutes.

Jobs may process on multiple GPUs simultaneously, so GPU minutes don't directly match wait time.

## Chapter 8: Vary Region

Use Vary Region editor to regenerate specific parts of upscaled images.

Appears after upscaling. Results guided by original image content and selected area.

Compatible with V5, V6, Niji 5, and Niji 6.

### How to Use Vary Region

1. Generate image with `/imagine`
2. Upscale with `U` buttons
3. Click `🖼️ Vary (Region)` to open editor
4. Select freehand or rectangular tools
5. Choose areas to regenerate
   - Larger selections = more creative changes
   - Smaller selections = subtle changes
   - Note: Can't edit selections, but can undo steps
6. Click `Submit →` to send request
   - Note: Can use multiple times with different selections
7. View results in new image grid

### Vary Region Examples

- Upscaled Image: colorful candy brooches
  - Selection → Result

- Upscaled Image: architectural drawing of a house
  - Selection → Result

### Vary Region + Remix Mode

Use Remix mode to update prompts while regenerating areas.

- Original: `gouache alligator in sunglasses`
- Selection
- Updated prompt: `gouache alligator in green sunglasses`
- Result

### Technical Details

Vary (Region) jobs honor these parameters:
--chaos, --fast, --iw, --no, --stylize, --relax, --style, --version, --video, --weird

## Chapter 9: Vary Region + Remix

Combine Remix Mode and Vary Region editor to regenerate parts using modified prompts.

Appears after upscaling. Results guided by original image, selection area, and modified prompt.

Compatible with V5, V6, Niji 5, and Niji 6.

### How to Use Remix Mode with Vary (Region)

1. Enable Remix Mode in `/settings`
2. Generate image with `/imagine`
3. Upscale with `U` buttons
4. Click `🖼️ Vary (Region)` to open editor
5. Select areas to regenerate
   - Larger selections = more creative changes
   - Smaller selections = subtle changes
   - Note: Can't edit selections, but can undo steps
6. Modify your prompt
   - Focus on selected area details
   - Be direct: "meadow stream" not "Please change the trail"
7. Click `Submit →` to send request
   - Note: Can experiment with different selections and prompts
8. View results in new image grid

### Next Steps

You can upscale varied images and use Vary Region again.

Examples:
- Selection → Result
- Updated prompt: "hot air balloon lithograph" → Result
- Updated prompt: "castle in the meadow lithograph" → Result

### Prompting Tips

**Selections**: Size affects outcome. Larger = more context but potential unwanted changes. Smaller = more control.

**Prompts**: Focus on desired changes in selected areas. Shorter, focused prompts often work better.

**Workflow**: Change multiple parts by working one area at a time.

### Examples

1. Original: `gouache alligator in sunglasses`
   - Selection
   - Updated: `gouache alligator in green sunglasses`
   - Result

2. Original: `scratchboard apple tree branch`
   - Selection
   - Updated: `rainbow scratchboard apple tree branch`
   - Result

### Technical Details

Vary (Region) + Remix jobs honor:
--chaos, --fast, --iw, --no, --stylize, --relax, --style, --version, --video, --weird

### Multi Prompts

Blend concepts with `::` separator

**Basics**

`::` separates prompt parts for individual consideration  
No space between colons

Examples:  
`space ship` (single concept: sci-fi spaceship)  
`space:: ship` (separate concepts: sailing ship in space)  
`cheese cake painting` (single concept: painted cheesecake)  
`cheese:: cake painting` (separate: painted cakes made of cheese)  
`cheese:: cake:: painting` (three separate concepts)

**Prompt Weights**

Add number after `::` to assign relative importance

Examples:  
`space:: ship` (equal weight)  
`space::2 ship` (space twice as important)

Weights normalize:  
`space:: ship` = `space::1 ship` = `space::100 ship::100`  
`cheese::2 cake` = `cheese::4 cake::2` = `cheese::100 cake::50`

Model version compatibility varies for decimal weights

**Negative Prompt Weights**

Remove unwanted elements with negative weights  
Sum must remain positive

Example:  
`still life gouache painting` (various objects)  
`still life gouache painting:: fruit::-.5` (fewer fruits)

**The --no Parameter**

`--no` equals `-0.5` weight in multi-prompt  
`vibrant tulip fields:: red::-.5` = `vibrant tulip fields --no red`

## Chapter 63: Fast, Relax, and Turbo Modes

Midjourney uses GPUs to process prompts. Subscription plans determine how much GPU time you get.

**Fast Mode**  
Default setting. Uses your monthly GPU time to provide near-instant access to a GPU.

**Relax Mode**  
Available to Standard, Pro, and Mega subscribers. Unlimited image generation, but jobs queue behind others. Wait times are typically 1–10 minutes and reset monthly.

**Turbo Mode**  
Experimental high-speed mode. Up to 4x faster than Fast, but uses 2x the GPU minutes. Available only for Midjourney Model Versions 5, 5.1, 5.2, and 6. Falls back to Fast if unavailable or incompatible.

---

### GPU Minute Costs

Average `/imagine` job = ~1 GPU minute.

Factors affecting time:

| Factor | Lower Time | Average Time | Higher Time |
|--------|------------|--------------|-------------|
| **Job Type** | Variations | `/imagine` | Upscale |
| **Aspect Ratio** | Default (square) | Tall or wide | - |
| **Quality Parameter** | `--q 0.25` or `--q 0.5` | `--q 1` (default) | `--q 2` (legacy models) |
| **Stop Parameter** | `--stop 10` | `--stop 99` | `--stop 100` (default) |

Use `/info` before and after to see GPU minute usage.

---

All users have access to Fast and Turbo modes. Only Standard, Pro, and Mega plan subscribers can use Relax mode.
